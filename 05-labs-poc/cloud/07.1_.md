## Phase 3 — Web Application Access

### How IIS Routes Requests

```
Browser → Security Group (port check)
       → Windows Firewall (port check)
       → IIS (W3SVC) — matches hostname to site binding
       → Physical path: C:\inetpub\vhosts\<production-domain>\httpdocs\
       → Application Pool → runtime (ASP.NET / PHP)
       → Response
```

The binding (`*:443:<production-domain>`) is the equivalent of nginx's `server_name`. IIS only responds to hostnames that have a configured binding on the target site.

### Option A — hosts File Override (Zero Config, Recommended for Quick Testing)

On your **local Mac**, override DNS for your machine only — production DNS and all other users are unaffected:

```bash
sudo nano /etc/hosts
```

Add:
```
<test-elastic-ip>    <production-domain> www.<production-domain>
```

Flush DNS cache:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Open browser → `https://<production-domain>` → loads from test instance with full production data.

To revert after testing:
```bash
sudo nano /etc/hosts   # remove the added line
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

### Option B — Staging Subdomain Binding (Persistent Testing URL)

Add IIS bindings to route `<project-name>.staging.<root-domain>` → `<production-domain>` IIS site:

```powershell
# HTTP binding
New-WebBinding -Name "<production-domain>" -Protocol "http" -Port 80 -HostHeader "<project-name>.staging.<root-domain>"

# HTTPS binding — attach existing production SSL cert
New-WebBinding -Name "<production-domain>" -Protocol "https" -Port 443 -HostHeader "<project-name>.staging.<root-domain>" -SslFlags 1
$cert = Get-Item Cert:\LocalMachine\My\<ssl-cert-thumbprint>
(Get-WebBinding -Name "<production-domain>" -Protocol "https" -HostHeader "<project-name>.staging.<root-domain>").AddSslCertificate($cert.Thumbprint, "My")
```

List available certs and their thumbprints:
```powershell
Get-ChildItem Cert:\LocalMachine\My | Select-Object Subject, Thumbprint, NotAfter
```

Add DNS A record:
```
A    <project-name>.staging    →    <test-elastic-ip>    TTL 300
```

> Browser will show a cert mismatch warning since the cert covers `<production-domain>` not the staging subdomain — click Advanced → Proceed for testing purposes.

### 4.4 — Whitelist Your Test Machine IP

> **Why this matters:** Your ISP/office IP may be on Spamhaus/SpamCop blacklists. Without whitelisting, every inbound test connection gets rejected with `554 The IP address was found in a DNS blacklist`.

Get your public IP:
```bash
curl -s https://api.ipify.org
```

Add to whitelist on server (PowerShell):
```powershell
Add-Content "C:\Program Files (x86)\Mail Enable\Config\SMTP-WHITELIST.SYS" "<your-public-ip>"
```

Enable whitelist in MailEnable Admin:
```
SMTP → Properties → Whitelist tab → ✅ Enable whitelist → Apply → OK
```

### 4.5 — DKIM Key Generation

Run in **PowerShell as Administrator**:

```powershell
$domain = "<project-name>.staging.<root-domain>"

# Generate RSA 2048-bit key pair
$rsa = New-Object System.Security.Cryptography.RSACryptoServiceProvider(2048)
$privateKeyBytes = $rsa.ExportCspBlob($true)
$privateKeyB64 = [Convert]::ToBase64String($privateKeyBytes, 'InsertLineBreaks')
$privateKeyPem = "-----BEGIN RSA PRIVATE KEY-----`r`n$privateKeyB64`r`n-----END RSA PRIVATE KEY-----"
$privateKeyPem | Out-File "C:\Program Files (x86)\Mail Enable\Config\DKIM\default-$domain.key" -Encoding ASCII

# Create DKIM config file
$dkimConfig = "<BASEELEMENT><ELEMENT><TYPE>Selector</TYPE><SELECTORNAME>default</SELECTORNAME><DNS-TXT>v=DKIM1; </DNS-TXT></ELEMENT><ELEMENT><TYPE>Options</TYPE><ACTIVESELECTOR>default</ACTIVESELECTOR><KEYFILE>default-$domain.key</KEYFILE><SIGN>1</SIGN><HASHALGORITHM>rsa-sha256</HASHALGORITHM><HEADERCANONICALIZATION>relaxed</HEADERCANONICALIZATION><BODYCANONICALIZATION>relaxed</BODYCANONICALIZATION><LIMITBODYHASHLENGTH>-1</LIMITBODYHASHLENGTH><INCLUDEUSERIDENTITY>0</INCLUDEUSERIDENTITY></ELEMENT></BASEELEMENT>"
$dkimConfig | Out-File "C:\Program Files (x86)\Mail Enable\Config\DKIM-$domain.SYS" -Encoding ASCII

Write-Host "DKIM files created"
```

Extract public key for DNS:

```powershell
$domain = "<project-name>.staging.<root-domain>"
$rsa2 = New-Object System.Security.Cryptography.RSACryptoServiceProvider
$rawKey = (Get-Content "C:\Program Files (x86)\Mail Enable\Config\DKIM\default-$domain.key" -Raw)
$b64 = $rawKey -replace "-----BEGIN RSA PRIVATE KEY-----|-----END RSA PRIVATE KEY-----" -replace "\s",""
$rsa2.ImportCspBlob([Convert]::FromBase64String($b64))
$params = $rsa2.ExportParameters($false)

function Encode-DERLength($len) {
    if ($len -lt 128) { return [byte[]]@($len) }
    elseif ($len -lt 256) { return [byte[]]@(0x81, $len) }
    else { return [byte[]]@(0x82, [byte](($len -shr 8) -band 0xFF), [byte]($len -band 0xFF)) }
}
function Encode-DERInteger($bytes) {
    if ($bytes[0] -band 0x80) { $bytes = @([byte]0x00) + $bytes }
    return @([byte]0x02) + (Encode-DERLength $bytes.Length) + $bytes
}
$modInt = Encode-DERInteger $params.Modulus
$expInt = Encode-DERInteger $params.Exponent
$seqContent = $modInt + $expInt
$sequence = @([byte]0x30) + (Encode-DERLength $seqContent.Length) + $seqContent
Write-Host "p=$([Convert]::ToBase64String([byte[]]$sequence))"
```

Add output to DNS:
```
Type:  TXT
Name:  default._domainkey.<project-name>.staging
Value: v=DKIM1; k=rsa; p=<output from above>
TTL:   600
```

## Phase 7 — Cutover Day Sequence

```
1. MGN → Launch cutover instance  (fresh snapshot of latest prod data)
2. EC2 → Associate <production-elastic-ip> to cutover instance
3. Plesk → retrieve new/renewed license:
   & "C:\Program Files (x86)\Plesk\bin\plesk.exe" bin license --retrieve
4. MailEnable → redo SMTP Connector config (General tab — update hostname to prod mail domain)
5. Update PTR on <production-elastic-ip> to production mail hostname
6. DNS → flip all A/MX/SPF records from <on-prem-ip> to <production-elastic-ip>
7. Monitor SMTP logs live:
   Get-Content "C:\Program Files (x86)\Mail Enable\Logging\SMTP\SMTP-Activity*.log" -Wait -Tail 50
8. Send test mail from each production domain → confirm Gmail delivery
9. Send from Gmail → each production domain → confirm inbound
10. Keep on-prem live for 48 hours as emergency fallback
11. Raise DNS TTLs back to 3600 after 48hrs stable
12. Decommission on-prem after 48hrs confirmed stable
```

### ⚠️ IIS Binding to Fix at Cutover

One IIS site has the on-prem IP hardcoded as its site name and binding. Update this to `<production-elastic-ip>` in IIS Manager before flipping DNS.

### ⚠️ Plesk SSL Certificate

Plesk's own SSL certificate may be expired. Renew before cutover to avoid SSL warnings on the Plesk panel.
